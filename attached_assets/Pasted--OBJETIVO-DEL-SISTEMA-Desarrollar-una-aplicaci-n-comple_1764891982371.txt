✔ OBJETIVO DEL SISTEMA

Desarrollar una aplicación completa en Python que gestione ventas, inventario, cajas, traspasos entre tiendas, devoluciones, reportes automáticos, alertas, múltiples métodos de pago y manejo de múltiples tiendas naturistas con independencia de inventarios y cajas.

✔ ALCANCE

La aplicación debe funcionar para varias tiendas (ejemplo: Tunal, 20 de Julio).
Cada tienda opera de manera independiente en inventario y cajas menores.
Las cajas mayores digitales se comparten entre todas las tiendas.

La aplicación debe ser escalable para añadir nuevas tiendas y usuarios.

6. TRANSFERENCIAS ENTRE CAJAS

Debe permitir transferir entre:

Caja mayor ⇆ caja menor
Entre métodos
Entre cajas mayores globales
Reglas:

Caja mayor efectivo: independiente por tienda

Cajas mayores Nequi, Daviplata, Bold: globales entre tiendas

No se puede transferir efectivo entre tiendas por cajas cruzadas

Ejemplos válidos:

Caja menor Nequi (20 de Julio) → Caja mayor Nequi (Global)

Caja mayor Bold (Global) → Caja menor efectivo (Tunal)

Caja mayor Daviplata (Global) → Caja mayor Nequi (Global)

Registro obligatorio:

Usuario
